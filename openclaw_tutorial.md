## ขั้นตอนที่ 1: ตั้งค่าให้ Bot ตอบข้อความในกลุ่ม Telegram


### Prompt ให้ AI ทำ


> ปรับ Telegram Group Policy เป็น `open` และตั้ง `allowFrom` ให้รับจากทุกคน (`"*"`)
> จากนั้น restart gateway เพื่อให้การตั้งค่ามีผล


### ผลลัพธ์ที่ควรได้ใน `openclaw.json`


```json
{
  "channels": {
    "telegram": {
      "groupPolicy": "open",
      "allowFrom": ["*"]
    }
  }
}
```


---


## ขั้นตอนที่ 2: ตั้งค่า BotFather และเพิ่ม Bot เข้ากลุ่ม


ต้องตั้งค่าผ่าน BotFather **ก่อน** เพิ่ม Bot เข้ากลุ่ม


1. เปิด BotFather → เลือก Bot ของเรา
2. ตั้งค่า **Allow Groups** → `Enabled`
3. ตั้งค่า **Group Privacy** → `Disabled` (จำเป็นเพื่อให้ Bot อ่านข้อความในกลุ่มได้)
4. เพิ่ม Bot เข้ากลุ่ม Telegram ที่ต้องการ


### ผลลัพธ์ที่ควรได้


- Bot สามารถตอบในกลุ่ม Telegram ได้เมื่อมีการ mention @ชื่อบอท


---


## ขั้นตอนที่ 3: สร้าง Agent ตัวใหม่


สร้าง Agent เพิ่มอีกตัวแยกจาก Agent หลัก เพื่อให้แต่ละ Agent ทำหน้าที่เฉพาะทางได้


### Prompt ให้ AI ทำ


> สร้าง Agent ใหม่ชื่อ `content` ด้วยคำสั่ง `openclaw agents add`


### คำสั่งที่ AI จะรัน


```bash
openclaw agents add <ชื่อ Agent>
```


### ผลลัพธ์ที่ควรได้


- มีโฟลเดอร์ใหม่ที่ `~/.openclaw/workspace-<ชื่อ Agent>/`
- ภายในมีไฟล์ `bootstrap.md` สำหรับตั้งค่า System Prompt ของ Agent ตัวนี้


---


## ขั้นตอนที่ 4: เพิ่ม Topics ในกลุ่ม Telegram


Topics ทำให้แต่ละหัวข้อในกลุ่มใช้ System Prompt คนละตัวได้ — เหมือนมีหลาย Agent ในกลุ่มเดียวกัน


### วิธีทำ


1. สร้าง Topic ในกลุ่ม Telegram (ผ่านแอป Telegram)
2. ขอ Group ID และ Topic ID ทั้งหมดพร้อมชื่อจาก AI โดยการแชท


---


## ขั้นตอนที่ 5: Bind กลุ่ม Telegram กับ Agent ตัวใหม่


ผูกกลุ่ม Telegram กับ Agent Workspace ที่ต้องการ เพื่อให้กลุ่มนั้นใช้ Agent เฉพาะทางตัวใหม่


### Prompt ให้ AI ทำ


> - Bind กลุ่ม Telegram `<Group ID>` กับ Agent `<ชื่อ Agent>` ที่ `~/.openclaw/workspace-<ชื่อ Agent>`
> - restart gateway หลัง bind เสร็จ


### ตัวอย่างผลลัพธ์ใน `openclaw.json`


```json
{
  "bindings": [
    {
      "agentId": "content",
      "match": {
        "channel": "telegram",
        "accountId": "default",
        "peer": {
          "kind": "group",
          "id": "<Group ID>"
        }
      }
    }
  ]
}
```


### วิธีทดสอบ


- ส่งข้อความหา Bot ในกลุ่ม → ต้องตอบเหมือนไม่เคยรู้จักกันมาก่อน (เพราะเปลี่ยน Agent แล้ว)


---


## ขั้นตอนที่ 6: ตั้ง Prompt ของแต่ละ Topic ให้แตกต่างกัน


### Prompt ให้ AI ทำ


> ตั้งค่า Telegram Topics สำหรับกลุ่ม `<Group ID>` ตาม System Prompt ด้านล่าง แล้ว restart gateway หลังเสร็จสิ้น


#### Topic `<Topic ID>` — ค้นหาข่าว


System Prompt:


```text
คุณมีหน้าที่เป็นนักวิจัยข่าว


งานของคุณ:
- ใช้ Brave Search หาข่าวเกี่ยวกับหัวข้อที่ผู้ใช้ระบุในช่วง 7 วันที่ผ่านมา
- คัดเฉพาะข่าวที่เป็นข่าวจริงจากแหล่งที่น่าเชื่อถือ
- หลีกเลี่ยงหน้ารวมข่าว หน้าหมวดหมู่ และหน้าแรกของเว็บไซต์ ถ้ายังไม่มีข่าวรายชิ้น
- พยายามหาข่าวที่หลากหลาย
- รวมลิงก์ข่าวทั้งหมดที่เกี่ยวข้องและไม่ซ้ำ
- จัดอันดับข่าวที่สำคัญที่สุด 5-10 ข่าว
- สำหรับข่าวแต่ละข้อ ต้องมี:
  1) พาดหัวข่าว
  2) แหล่งข่าว
  3) วันที่
  4) สรุป 1-2 ประโยค
  5) เหตุผลว่าทำไมน่าสนใจ
  6) ลิงก์ข่าวแบบ URL เต็ม วางใต้ข่าวข้อนั้นทันที
- ถ้าจำเป็นต้องมีส่วนสรุปท้ายข้อความ ให้เป็นสรุปภาพรวมสั้น ๆ เท่านั้น ไม่ใช่รายการลิงก์ซ้ำ


กติกา:
- อย่าเขียนบทความยาว
- อย่าแต่งข้อมูลเพิ่ม
- ถ้าไม่มั่นใจวันที่หรือข้อเท็จจริง ให้ระบุว่าไม่ชัวร์
- ตอบเป็นภาษาไทย


เมื่อทำงานเสร็จแล้ว คุณต้องส่งผลลัพธ์กลับมาในแชท topic นี้เสมอ
อย่าหยุดแค่คิดหรือทำงานภายใน session อย่างเดียว
โดยคำตอบสุดท้ายต้องใส่ลิงก์ไว้ใต้ข่าวแต่ละข้อทันทีตามรูปแบบที่กำหนด
```


#### Topic `<Topic ID>` — เขียนข่าว


System Prompt:


```text
คุณมีหน้าที่เป็นนักเขียนคอนเทนต์ข่าวบน Social Media


งานของคุณ:
- รับลิงก์ข่าวจากผู้ใช้
- เข้าอ่านลิงก์ข่าวทีละลิงก์เพื่อดึงสาระสำคัญ
- สังเคราะห์ข่าวทั้งหมดให้กลายเป็นบทความ long-form content ภาษาไทย 1 ชิ้น
- เนื้อหาต้องอ่านง่าย ลื่น มีโครงเรื่อง ไม่ใช่แค่สรุปทีละข้อ เชื่อมโยงประเด็นต่าง ๆ ให้เป็นบทความเดียวด้วย อ่านสนุก แต่ไม่เวอร์
- ถ้ามีข้อจำกัดหรือข้อกังขาจากแหล่งข่าว ให้ระบุอย่างระมัดระวัง


โครงสร้างที่ต้องการ:
1) หัวข้อบทความ
2) hook เปิดเรื่อง
3) เนื้อหาหลักแบบ long-form หลายย่อหน้า
4) สรุป takeaway ตอนท้าย
5) ปิดท้ายด้วย "แหล่งอ้างอิง" เป็นรายการลิงก์ที่ใช้


กติกา:
- ห้ามแต่งข้อเท็จจริง
- ห้ามเขียนจากลิงก์อย่างเดียวโดยไม่อ่านเนื้อหา
- ถ้าลิงก์ไหนเข้าไม่ได้ ให้ข้ามและบอกว่าไม่ได้ใช้
- ตอบเป็นภาษาไทย


เมื่อทำงานเสร็จแล้ว คุณต้องส่งบทความ final กลับมาในแชท topic นี้เสมอ
โดยคำตอบสุดท้ายต้องเป็นบทความที่พร้อมอ่านในแชท พร้อมแหล่งอ้างอิงท้ายข้อความ
```


### หลังตั้งค่า


- ตรวจสอบ `openclaw.json` ว่า Prompts ถูกเพิ่มเข้าไปถูกต้อง


### โครงสร้าง Topics ใน `openclaw.json`


```json
{
  "channels": {
    "telegram": {
      "groups": {
        "<Group ID>": {
          "topics": {
            "<Topic ID 1>": {
              "systemPrompt": "prompt สำหรับ topic แรก"
            },
            "<Topic ID 2>": {
              "systemPrompt": "prompt สำหรับ topic ที่สอง"
            }
          }
        }
      }
    }
  }
}
```

## ขั้นตอนที่ 7: เพิ่มความปลอดภัย — จำกัดกลุ่มและผู้ใช้


เมื่อทุกอย่างทำงานได้แล้ว ควรล็อกสิทธิ์ให้เฉพาะกลุ่มและผู้ใช้ที่อนุญาตเท่านั้น


### หา Group ID และ User ID ก่อน แล้ว Prompt ให้ AI ทำ


> - ปรับ `groupPolicy` เป็น `allowList` (จำกัดเฉพาะกลุ่มที่อนุญาต)
> - เพิ่ม Group ID `<Group ID>` ใน allowList
> - ปรับ `channels.telegram.groupAllowFrom` ให้มี User ID `<User ID>`
> - ปรับ `channels.telegram.allowFrom` ให้มี User ID `<User ID>`
> - ปรับ `requireMention` สำหรับกลุ่มนี้เป็น `false` (ไม่ต้อง mention Bot ก็ตอบได้)
> - restart gateway หลังปรับ setting


### หลังตั้งค่า


- ตรวจสอบว่า Agent สามารถตอบในกลุ่ม Telegram ได้โดยไม่ต้อง mention


---


## ขั้นตอนที่ 8: ให้ Agent สื่อสารข้ามกันได้


เปิดให้ Agent แต่ละตัวมองเห็น session ของกันและกัน และสามารถสื่อสารข้าม Agent ได้


### Prompt ให้ AI ทำ


> - ตั้งค่า `tools.sessions.visibility` เป็น `all`
> - ตั้งค่า `tools.agentToAgent.enabled` เป็น `true`
> - restart gateway


### หลังตั้งค่า


- ตรวจสอบว่าพิมพ์หา Bot ใน Topic ได้โดยไม่ต้อง mention


---


## ขั้นตอนที่ 9: ทดสอบใช้งาน Agent ผ่าน Topic


### วิธีทดสอบ


1. **ค้นหาข่าว** — ส่งข้อความใน Topic ค้นหาข่าว ดูว่า Bot ค้นและส่งลิงก์มาให้ถูกต้อง
2. **เขียนข่าว** — ส่งข้อความใน Topic เขียนข่าว ดูว่า Bot แปลงเป็น Social Media Content จากข้อมูลในลิงก์ที่ให้ได้อย่างถูกต้อง
3. **ถ้า Bot ไม่ทำตาม System Prompt** — สร้าง session ใหม่ด้วยคำสั่ง `/new` เพื่อให้ Agent โหล
